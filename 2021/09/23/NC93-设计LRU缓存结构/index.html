NC93-设计LRU缓存结构

2021-09-23

前几天的每日一题感觉没啥记录的必要。。今天是牛客打卡 day6。。这次的题目主要使用 stl 实现一种数据结构，用好了其实也还行，就是我这笨比用不好（所以才来做个记录

题目链接：NC93 设计LRU缓存结构

  • 两种操作：

    • set：实现键值对的插入，相同的 key 需要将原来的覆盖，且有一定的容量大小限制。
    • get：获取 key 对应的键值，并更新缓存。

  • 根据这些操作选用什么样的 stl 呢？首先由于这个元素也是一个个键值对，我们会首先想到用 map ，但是 map 是有序的，所以想到用 unordered_map，用它来查询 key 所对应的 value，但是 unordered_map 内部的顺序是混乱的，所以必须还得有一个东西存放原序列，且容易进行删除和添加元素的操作，最好是 set 时从头部插，而超过容器大小时删除末尾的元素（最不常用）。这时候就很明显了，采用 list 非常合适。list 存该序列，unordered_map存 list 中元素的迭代器及其对应的 key，这样可以很快找到key对应的元素，且很方便更新缓存。
