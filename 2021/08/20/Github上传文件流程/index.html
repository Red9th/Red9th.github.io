Github上传文件流程

2021-08-20

1. 在github上创建一个新仓库

创建的时候主要是是否添加一个README.md文件，这个对最后push的步骤有点影响，后面详细说明。

2. 将文件夹变成git可管理的仓库

首先选择一个要上传的文件夹，并进入这个文件夹，右键单击Git Bash Here。

然后输入以下命令：

$ git init

这是这个文件夹下会产生一个.git文件，一般是隐藏起来的，点击查看，勾选隐藏文件就可以看到了，这是这个文件夹就变成git可管理的仓库了。

3. 提交要上传的文件

将这个文件夹下的所有文件全部提交：

$ git add .

指定该文件夹下的文件提交：

$ git add 文件名

然后输入：

$ git status

看一下当前分支是否在main上，因为现在github的默认分支是main，如果开始在master上，后面就会创建一个master分支，提交的文件也不在main上了。如果不在main上的话，就输入下面的命令：

$ git checkout -b main

这样，就转到main分支上了。

4. 提交注释

$ git commit -m "注释"

5. 连接远程仓库

  1. 首先到C:\Users\HP这个路径下看有没有.ssh文件，我的电脑是惠普的所以是HP，其他电脑可能是别的名字，根据自己电脑来。如果有的话，看下里面有没有id_rsa和id_rsa.pub这两个文件，有就不用管这一步了。没有的话就继续下一步创建SSH密匙。

  2. 在电脑任意处右键单击Git Bash Here

  3. 设置用户名

    $ git config --global user.name "你的名字"

  4. 设置用户名邮箱

    $ git config --global user.email "你的邮箱"

  5. 查看设置

    $ git config --list

  6. 创建SSH密匙，这一步一路回车即可。

    $ ssh-keygen -t rsa -C "你刚刚输入的邮箱"

  7. 配置github

    • 点击头像旁边的倒三角符号，单击settings，进入SSH and GPG keys，点击New SSH key，Title随便输入什么，下面的Key填C:\Users\HP\.ssh下的id_rsa.pub这个文件中的内容，直接复制粘贴即可。
    • 点击Add SSH key

6. 关联本地仓库

$ git remote add origin 你的SSH

7. 将内容推送到远程仓库上

如果创建仓库的时候添加了md文件，显然本地仓库和远程仓库的文件不匹配，需要先合并一下：

$ git pull --rebase origin main

再输入：

$ git push origin main

如果没有添加md文件，那么一开始这个仓库就是空的，直接输入：

$ git push origin main

这样就push上去了，你也可以在github上面看到刚刚提交的文件了。

等远程仓库里面有了内容之后，下次再从本地库上传内容的时候只需输入下面命令就可以了：

$ git push origin master
