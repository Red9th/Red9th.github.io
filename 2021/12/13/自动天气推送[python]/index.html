Red9th
2021-12-13
杂

一个鸽了一学期的小项目，终于在学期末结题了。

一、写脚本

其实脚本的撰写没有花费很长时间，几天就写好并且测试完毕了，就是代码还是不够规范。

思路和学习 python 爬虫的时候跟老师带着写的豆瓣爬虫项目其实差不多。无非是我是爬取天气，而豆瓣爬虫是在爬取上面的电影信息。

1.1 困境

虽然很快就完成了脚本，但是开始写脚本之前还是遇到了非常多的问题，在豆瓣爬虫的时候，由于豆瓣 top250 这个网页比较老，反爬机制很简单，在 headers 下面改一下马甲就可以轻松爬取到电影信息，但是我要爬取的天气网站（当时看的有腾讯天气、墨迹天气等等）反爬机制非常搞新手心态，f12 打开开发者页面，几乎全是动态 js 元素，我不由得陷入沉思，要不要继续学习能够轻松爬取动态网页的 python 库呢？后来其实没有，一来开学了，时间不怎么够，拖延症又厉害，二来在 github 上发现了一个和我的项目类似的项目，参考了一下他的代码，发现了另外一个天气网站，这个网站的动态元素较少，每个城市都对应一个网页，很棒，然后就开始在它的源代码上进行捣鼓。

1.2 困境之二

虽然这个网页的动态元素很少，但是到了我要爬取的天气信息上就突然消失了，当时我到处找找不到，最后只找到了一个不知道在哪里显示了的几句话，它有几个小时更新一次的时间和天气信息，然后从这里就开始了。

二、写配置文件

之前说过参考了那个类似项目，看到他用 github actions 跑的，我就也想使用 github actions 每天定时自动运行我的天气脚本。

2.1 困境之三

发现关于 github actions 配置 python 脚本的相关内容非常少，对英文文档又处于极度懵逼的状态，然后这个项目就从开学一直鸽到现在快期末（。

只能说，幸好还是多多百度了一下（后来可能换了一个关键字就找到了一个 shell 脚本的部署和一个 python 脚本的部署的博客）参考了一下，又回去看了看官方文档，终于搞明白了 github actions 的原理和相关配置文件的撰写，这个东西确实非常的强大。然后自己写了配置文件，到这里也就花了一天的时间。
