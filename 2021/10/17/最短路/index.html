Dijkstra

  • 每次找一个 st 为 false 的点，且该点的 dis 是所有 st = false 中的最小。
  • 用该点去更新其他的点。
  • 更新完毕之后，该点的 dis 确定，st 为 true。

朴素dij

适用于m 和 n ^ 2 一个级别：稠密图
时间复杂度：O(n ^ 2)

Bellman-Ford

适合于有边数限制的最短路，也可以判断负环，但一般不用，主要用 spfa 来判断负环。

算法思路比较简单，需要限制多少条边就迭代多少次，每次用上次迭代的结果来更新所有的边（这就是 bk 数组的含义，避免因为当前的迭代更新的某些点又去更新另外的点，达不到边数限制的作用）。

最后为什么要dis[n] > 0x3f3f3f3f / 2呢？假设 n 不能到达，而由于最后一次迭代可能会让 dis[n] 改变（某个点与 n 相连且边权为负），这样就会比 0x3f3f3f3f 要小，如果只判断 dis[n] == 0x3f3f3f3f 的话就会出问题。

Bellman - ford 算法判断负环的思路是：在迭代 n - 1 次后如果还可以进行松弛操作，说明一定存在负环。如果采用队列实现，那么当某个结点入队了 n 次时可以判断出存在负环。（有负环可以一直更新下去）

SPFA

这个算法的思路和 dij 极为相似，但是又略有不同：

二者整体框架相同，但是 dij 的 st 在等于 true 之后不会改变（传统写法，y 总教的和传统的有差异，基本和 spfa 相同），因此后面也不会被更新，如果出现负权就会出问题。而 spfa 的 st 是在随时变化的，只要出队 st 值就更新为 false，只要入队就更新为 true。其余代码几乎一模一样。

同时，spfa 也可以不需要 st 这个东西，这样也可以达到反复入队的效果。拿上面堆优化的 dij 稍改之后去交下面这个题，完全可以过。

852. spfa判断负环

  • 开始不需要将 dis 数组赋值为 INF，可以想象一个虚拟源点，该源点到图中每个节点的距离都是 0，然后再从这些节点依次拓展，所以开始的值实际上无关紧要。
  • cnt[j] 表示 1 ~ j 上最短路的边数。
  • cnt[j] = cnt[a] + 1，含义是：若 j 能够被 a 更新（即出现负权边），那么边数一定是在 cnt[a] 的基础上加 1 的且一定存在负权边。
  • if(cnt[a] >= n) return true; 判断是否有负环，原理是：如果 cnt[a] 大于等于 n，说明 1 ~ a 上经过了大于等于 n + 1 个点，而根据抽屉原理，其中必定有重复的点，即产生了环，由于题中只有负权才会让边更新（因为开始全部初始化为同一个值），所以一定存在负环。
  • 如果是判断 1 - n 之间的路径有无负环：先求所有强联通分量，然后缩点，之后就变成了拓扑图了。然后依次用 spfa 算法判断每个强联通分量中是否有负环，然后从起点到终点递推一下，求出从起点到每个点的所有路径中，是否存在负环即可。（或者仅把源点推入，应该是可行的吧？）（暂时还不会 OTZ，这里是摘的 y 总的和其他聚聚的评论）

SFL优化

当 spfa 被卡的时候，就必须使用优化手段了，就比如下面这个题目。直接上裸的板子的话，势必会 TLE ，这时就需要 SFL 优化了。
那么什么是 SFL 优化呢？
SFL（Small Table First）优化思路是将原队列改成双端队列（deque），对要加入队列的点 p，如果 dis[p] 小于队头元素 u 的 dis[u]，将其插入到队头，否则插入到队尾。
